## Nightly Run 23152001787 Recovery

Date: 2026-03-16

Task ID: `DV-02-T02`

## Summary
Recovered GitHub Actions nightly run `23152001787` by fixing one Windows packaging regression and two cross-platform build regressions that blocked the matrix before release assembly:

- Windows MSI generation failed because the PowerShell WiX wrapper treated native flags like `-out` as function parameters.
- Linux nightlies no longer installed the OpenGL development headers required by `rend_gl`.
- macOS `sgame` builds used `std::from_chars(float)`, which is not available on the hosted libc++ toolchain used by GitHub Actions.

## Failures Observed

### Windows
Job `67256215267` failed in `Build MSI installer` with:

```text
Parameter cannot be processed because the parameter name 'out' is ambiguous.
Possible matches include: -OutVariable -OutBuffer.
```

The immediate cause was `tools/build_msi.ps1`. `Invoke-NativeCommand` was declared as an advanced PowerShell function and received native arguments through `ValueFromRemainingArguments`, so bare WiX switches such as `-out` were intercepted by PowerShell before they could reach `candle.exe` or `light.exe`.

### Linux
Job `67256215280` failed during `Build (Linux/macOS)` while compiling the OpenGL renderer:

```text
../src/rend_gl/qgl.h:40:10: fatal error: GL/gl.h: No such file or directory
```

The nightly Ubuntu dependency install path no longer included the package that provides `GL/gl.h`.

### macOS
Job `67256215387` failed during `Build (Linux/macOS)` in multiple `sgame` translation units with:

```text
error: call to deleted function 'from_chars'
```

The failure came from float parsing in `src/game/sgame/commands/command_system.hpp`, and an additional `std::from_chars(float)` use existed in `src/game/sgame/gameplay/g_target.cpp`.

## Implementation

### WiX Native Invocation Hardening
Updated `tools/build_msi.ps1` so `Invoke-NativeCommand` now receives explicit `-FilePath` and `-Arguments` parameters. All `heat.exe`, `candle.exe`, and `light.exe` calls now pass string arrays instead of relying on PowerShell's remaining-arguments binding.

This prevents WiX switches such as `-out`, `-ext`, and `-cg` from being reinterpreted as PowerShell function parameters.

### Linux OpenGL Header Dependency Restore
Updated `.github/workflows/nightly.yml` to install `libgl1-mesa-dev` in the Linux dependency step. That package restores the standard OpenGL headers required by `src/rend_gl/qgl.h`.

### Portable Float Parsing For `sgame`
Replaced the `std::from_chars(float)` parsing path in `src/game/sgame/commands/command_system.hpp` with a strict `std::strtof`-based implementation that:

- copies the `std::string_view` into a null-terminated buffer
- resets `errno`
- requires full-string consumption
- rejects empty input

Applied the same portable float parsing approach in `src/game/sgame/gameplay/g_target.cpp` for `CS_SKYROTATE` parsing. Integer parsing there continues to use `std::from_chars`, which remains portable across the CI toolchains in use.

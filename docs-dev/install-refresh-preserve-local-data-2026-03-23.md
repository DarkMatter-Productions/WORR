# Install Refresh Preserve Local Data (2026-03-23)

## Summary

The default VS Code launch flow runs `meson: build`, which includes
`tools/refresh_install.py`. That refresh path previously deleted the entire
`.install/` tree before copying fresh binaries and repackaging `basew/pak0.pkz`.

For local development, users commonly keep their licensed Quake II data in
`.install/basew/` as `pak*.pak`. Deleting `.install/` on every refresh erased
those files, which caused the next client launch to fail early with missing base
assets such as `pics/colormap.pcx`.

## Root Cause

- `tools/stage_install.py` called `shutil.rmtree(install_dir)` unconditionally.
- The default launch task executes `meson: build`, which executes
  `install: refresh (.install)`.
- Refreshing `.install/` therefore removed user-supplied stock data under
  `.install/basew/`, even though that data is not generated by the build.

## Changes

- `tools/stage_install.py`
  - Stopped deleting the entire `.install/` root during refresh.
  - Added targeted cleanup for generated root runtime files only:
    - `worr_*`
    - `worr*.exe`
    - `worr*.dll`
    - `worr*.pdb`
    - `worr*.so`
    - `worr*.dylib`
  - Added targeted cleanup for generated `basew/` content only:
    - `cgame_*`
    - `sgame_*`
    - `shader_vkpt`
    - `pak0.pkz`
  - `copytree(..., dirs_exist_ok=True)` now refreshes generated build outputs
    while preserving user-managed local data such as:
    - `pak*.pak`
    - configs
    - saves
    - demos
    - screenshots
    - loose maps/mod data

## Validation

Recommended local validation:

```powershell
python tools/refresh_install.py --build-dir builddir-win --install-dir .install --base-game basew
```

After refresh:

- `.install/basew/pak0.pkz` is regenerated from repo `assets/`
- `.install/basew/cgame_x86_64.dll` and `.install/basew/sgame_x86_64.dll` are refreshed
- user-supplied `.install/basew/pak*.pak` remain in place

## User Impact

Once Quake II base data is copied into `.install/basew/`, the default VS Code
launch task can be reused without recopying those files after every build.

# Q2 AAS Generator Reachability Bridge

Date: 2026-06-16

Task IDs: `FR-04-T11`, `FR-04-T16`, `DV-07-T06`

## Summary

This slice enables the imported BSPC reachability and clustering pass for Quake II BSP inputs by adding a WORR-native Q2 trace bridge. `worr_q2aas` now produces nonzero reachability and clusters for the staged `mm-rage.bsp` smoke map.

The bridge is deliberately scoped to the standalone generator. It does not pull WORR's runtime collision subsystem into BSPC. Instead, it uses the Q2 BSP planes, nodes, leaves, leafbrushes, brushes, brushsides, models, and entity lump already loaded by the imported Q2 BSP reader.

## Implementation

- Added `worr_q2aas_q2trace.c/.h`, a tool-local Q2 BSP trace bridge:
  - point contents through the Q2 BSP node/leaf tree;
  - box traces through Q2 BSP nodes, leaves, and brushes;
  - Q2 brush-surface flags and texture names copied into BotLib trace surfaces;
  - Q2 inline model bounds for mover reachability analysis;
  - source BSP checksum calculation for the generated AAS header;
  - Q2 entity string forwarding for BotLib's BSP entity parser.
- Updated `be_aas_bspc.c`:
  - Q2 maps call `WorrQ2AAS_BeginQ2Reach` instead of the Q3-only `CM_LoadMap` path;
  - BotLib import callbacks route trace, point contents, entity data, and inline model bounds to the Q2 bridge while it is active;
  - Q3 maps retain the original collision-map path.
- Updated `bspc.c`:
  - `MAPTYPE_QUAKE2` now participates in reachability/clustering after `AAS_Create`;
  - retained Q2 BSP data is freed after the map's AAS file is written.
- Updated `map.c`:
  - Q2 BSP lumps remain loaded during AAS generation, instead of being freed immediately after `Q2_LoadMapFromBSP`.
- Updated validation summary output:
  - strict `--require-reachability` now passes when `reachabilitysize` and `numclusters` are nonzero;
  - travel-type counts are included in map summaries.

## Known Limits

- This is a first static-world Q2 reachability bridge. It proves BotLib reachability can run against Q2 BSP data, but it is not a full reference-map acceptance pass.
- Door, platform, elevator, teleporter, water, hazard, and objective route semantics still need map-by-map validation.
- Rocket-jump reachability is currently generated by inherited BSPC/BotLib logic; WORR still needs an explicit policy gate before runtime bots use those routes.
- The standalone generator now writes reachability, but the runtime BotLib/AAS loader and WORR bot navigation adapter are still future tasks under `FR-04-T12` and `FR-04-T14`.

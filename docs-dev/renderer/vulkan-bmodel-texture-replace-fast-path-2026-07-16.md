# Vulkan inline-BSP texture-replace fast path

Date: 2026-07-16

Task ID: FR-01-T15

## Outcome

The Vulkan renderer now has native specialized fragment pipelines for opaque,
unlightmapped inline-BSP faces that OpenGL renders with GLS_TEXTURE_REPLACE.
The path preserves the OpenGL result while avoiding the general entity fragment
program's receiver and fallback-lighting work. It is entirely in rend_vk;
Vulkan never redirects a draw through OpenGL.

The feature is controlled by archived, default-on
vk_bmodel_texture_replace:

- 1 selects the specialized native pipeline when its strict batch gate accepts
  the face.
- 0 keeps the complete native Vulkan entity pipeline, for regression and
  driver-workaround comparison.

## Eligibility and shader contract

Registration-time inline-BSP geometry marks non-transparent faces without an
authored lightmap as VK_ENTITY_VERTEX_TEXTURE_REPLACE. At submission, the
specialized pipeline additionally requires:

1. an opaque batch and an opaque source texture;
2. only TEXTURE_REPLACE and optional INTENSITY material flags;
3. no alpha-test, translucent, glow-map, entity-effect, or receiver material
   flags; and
4. a successfully created specialized pipeline.

The gate deliberately permits only the FULLBRIGHT, NO_SHADOW, and NO_DLIGHT
entity-lighting subset. The OpenGL-equivalent texture-replace branch returns
before reading those bits, so accepting them makes the path correctly
reachable with r_fullbright 1 without broadening the material contract. Any
other entity bit keeps the complete native pipeline.

VK_ENTITY_GPU_BMODEL_TEXTURE_REPLACE is a submission-only marker. The
specialized fragment shader samples the base material, applies optional
texture intensity, and then:

- applies the established entity fog formula in the 7,236-byte fog-aware
  module; or
- skips that formula in the 3,296-byte no-fog module.

The recorder chooses the no-fog module only when
VK_Shadow_HasActiveSurfaceFog() reports neither global nor height fog. Global
or height fog immediately selects the fog-aware companion on the next frame.
Sky-only fog does not affect opaque entity-surface fog and therefore does not
block the no-fog module. No batch, geometry, descriptor, draw count, or upload
layout changes at a fog transition.

VK_STATS entity_texture_replace_draws records the selected specialized draws;
entity_texture_replace_no_fog_draws records the no-fog subset.
analyze_renderer_perf.py reports mean, p50, and p95 for both counters.

## Visual validation

All capture work used a hidden 960x720 native surface, isolated home
directories, win_headless 1, disabled input/audio, and Vulkan validation.

| Matrix | Result |
| --- | --- |
| fr01_bmodel_instances_unlightmapped_manifest.json | Exact OpenGL/Vulkan output over 218,400 crop pixels: zero RGB error and a matching 179,742-pixel material probe at IoU 1.0. This run selects the no-fog module. |
| fr01_bmodel_instances_fog_manifest.json | New generated unlightmapped 37-instance global-fog fixture passes 218,400 crop pixels with maximum RGB error 1/1/1, mean absolute RGB error at most 0.030509, and zero pixels over threshold. |

The fog fixture is generated by
tools/renderer_parity/generate_bmodel_instance_fog_fixture.py and stages
maps/worr_fr01_bmodel_instances_fog.bsp as both a package member and loose
asset. Its Vulkan telemetry records 16 entity_texture_replace_draws and zero
entity_texture_replace_no_fog_draws per frame, proving the visual fog result
used the fog-aware module. The ordinary no-fog control records 16 of both
counters.

## Controlled timing

The current-binary paired capture used the same unlightmapped 37-instance
fixture, configuration, adapter, driver, validation setting, and 100
post-warmup samples per backend. Only vk_bmodel_texture_replace=0/1 changed.

| Vulkan metric | Disabled (0) | Enabled (1) | Change |
| --- | ---: | ---: | ---: |
| Opaque entity GPU p50 | 0.5055 ms | 0.1620 ms | -68.0% |
| Opaque entity GPU mean | 0.51276 ms | 0.16261 ms | -68.3% |
| Opaque entity GPU p95 | 0.507 ms | 0.165 ms | -67.5% |
| Opaque scene GPU p50 | 1.171 ms | 0.824 ms | -29.6% |
| Completed GPU-frame p50 | 1.1995 ms | 0.851 ms | -29.1% |

The disabled row reports zero texture-replace counters. The enabled row
reports 16 texture-replace draws and 16 no-fog draws per sample. Both retain
18 Vulkan draws and 4,800 upload bytes per frame. The test system was Intel
Iris Xe Graphics with driver 31.0.101.5590 (2024-06-10), a 13th Gen Intel
Core i7-13700H, and Windows 11 Home 10.0.26200.

This is a focused entity-phase result. The stable paired world phase is nearly
unchanged (0.662 to 0.6585 ms p50), but this does not establish a
renderer-wide Vulkan/OpenGL GPU budget. The paired OpenGL completed-frame p50
is 0.318/0.316 ms and FR-01-T15 remains open.

## Reproduction

1. Generate and validate the fixture:

   python tools/renderer_parity/generate_bmodel_instance_fog_fixture.py --asset-root assets

   python tools/renderer_parity/generate_bmodel_instance_fog_fixture.py --asset-root assets --validate

2. Run the focused source and asset checks, regenerate SPIR-V, build, and
   refresh the install:

   python tools/renderer_parity/test_generate_bmodel_instance_fog_fixture.py

   python -m unittest tools.renderer_parity.test_vulkan_gpu_bmodel_submission_source tools.renderer_parity.test_analyze_renderer_perf

   python tools/test_package_assets.py

   python tools/gen_vk_world_spv.py --validate

   ninja -C builddir-win worr_vulkan_x86_64.dll

   python tools/stage_install.py --build-dir builddir-win --install-dir .install

   python tools/package_assets.py --assets-dir assets --install-dir .install

3. Run the ordinary and fogged matrices with Vulkan validation:

   python tools/renderer_parity/run_capture_matrix.py --install-dir .install --executable .install/worr_x86_64.exe --manifest assets/renderer_parity/fr01_bmodel_instances_unlightmapped_manifest.json --run-root .tmp/renderer-parity/fr01-bmodel-texture-replace-visual --vulkan-validation

   python tools/renderer_parity/run_capture_matrix.py --install-dir .install --executable .install/worr_x86_64.exe --manifest assets/renderer_parity/fr01_bmodel_instances_fog_manifest.json --run-root .tmp/renderer-parity/fr01-bmodel-texture-replace-fog --vulkan-validation

4. Capture the two performance controls with
   run_renderer_perf_capture.py, the same dense fixture and hardware metadata,
   launch cvars vk_bmodel_texture_replace=0 and
   vk_bmodel_texture_replace=1, then analyze each log pair with
   analyze_renderer_perf.py, capture.json, warmup 20, and min-samples 100.
